### （1）题目意思及生僻概念解释

**题目大意：**
在一个给定长宽的矩形棋盘上，有一个机器人站在左上角的起点。这个机器人的动作空间非常受限，每次只能执行两个动作之一：“向下走一格”或者“向右走一格”。任务是计算出从起点走到右下角终点，总共有多少种不同的行走路线。

**生僻与关键概念解释：**
* **$m \times n$ 网格：** 可以理解为一个二维矩阵或状态空间。$m$ 代表行数（上下方向），$n$ 代表列数（左右方向）。
* **不同路径：** 只要两条路线中哪怕只有一步的选择不同（例如一个是先右后下，另一个是先下后右），就算作两条独立的路径。
* **无障碍物：** 题目没有设置任何死胡同，只要不走出网格边界，任何向右或向下的移动都是合法的状态转移。

### （2）代码解题思路解析

这段代码采用的是**“自顶向下（Top-Down）的记忆化深度优先搜索（DFS）”**策略。从状态转移的角度来思考，逻辑如下：

1. **逆向思维（确定状态转移关系）：** 假设机器人已经站在了网格里的某一个坐标 $(i, j)$ 上。因为机器人只能向右或向下走，那么它到达 $(i, j)$ 的前置状态只有两种可能：
   * 从上方的格子 $(i-1, j)$ 向下走一步过来的。
   * 从左方的格子 $(i, j-1)$ 向右走一步过来的。
   因此，到达 $(i, j)$ 的总路径数，必然等于到达上方格子的路径数加上到达左方格子的路径数。这就构成了核心的递推关系：`dfs(i, j) = dfs(i-1, j) + dfs(i, j-1)`。

2. **确立边界条件（Base Case）：**
   * **越界：** 如果坐标 $i < 0$ 或 $j < 0$，说明退到了网格外面，这是一条不合法的路径，因此路径数为 0。
   * **原点：** 如果退回到了起点 $(0, 0)$，说明找到了一条完整的合法路线，路径数为 1。

3. **引入 `@cache` 消除重叠子问题：** 在纯粹的递归树中，很多坐标会被反复计算（例如从右边退回和从下面退回可能会访问同一个左上方的格子）。`@cache` 是一个装饰器，充当了“备忘录”的作用。一旦某个坐标 $(i, j)$ 的路径数被计算出来，就会被存入哈希表中。下次再查询同一个坐标时，直接以 $O(1)$ 的时间复杂度返回结果，从而将指数级的时间复杂度降维到多项式级别的 $O(m \times n)$。

### （3）附带注释的代码

```python
from functools import cache

class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        # 定义内部的递归函数，计算到达坐标 (i, j) 的路径总数
        @cache  # 缓存装饰器，避免重复计算 dfs 的结果（一行代码实现记忆化）
        def dfs(i: int, j: int) -> int:
            # 边界条件 1：如果坐标越界（处于网格左侧或上方），说明此路不通
            if i < 0 or j < 0:
                return 0
            # 边界条件 2：如果正好退回到了起点 (0, 0)，说明找到了一条有效路径
            if i == 0 and j == 0:
                return 1
            # 状态转移：当前格子的路径数 = 上方格子的路径数 + 左方格子的路径数
            return dfs(i - 1, j) + dfs(i, j - 1)
        # 启动计算，目标是网格的右下角，因为索引从 0 开始，所以坐标是 (m-1, n-1)
        return dfs(m - 1, n - 1)
```

### （4）每一行代码的详细解释

* `class Solution:`
    * 定义一个符合题目提交规范的类封装。
* `def uniquePaths(self, m: int, n: int) -> int:`
    * 定义主函数，接收网格的行数 $m$ 和列数 $n$，并声明返回值为整数形式的总路径数。
* `@cache`
    * Python 内置的函数装饰器。它的作用是自动记录其下方修饰的函数（即 `dfs`）的输入参数和对应的输出结果。当相同的参数再次被调用时，直接返回缓存结果，不执行函数体。
* `def dfs(i: int, j: int) -> int:`
    * 定义深度优先搜索的辅助函数，参数 $i$ 表示当前所在的行索引，$j$ 表示当前所在的列索引。
* `if i < 0 or j < 0:`
    * 检查当前坐标是否超出了网格的左边界或上边界。
* `return 0`
    * 如果越界，说明这种倒退方式是不合法的（现实中机器人不可能从墙外走进来），向上一层返回路径数 0。
* `if i == 0 and j == 0:`
    * 检查当前坐标是否正好是起点位置 $(0, 0)$。
* `return 1`
    * 如果到达起点，说明成功回溯出了一条完整的路径，向上一层提供 1 种路径基数。
* `return dfs(i - 1, j) + dfs(i, j - 1)`
    * 核心递归调用。分别向当前坐标的正上方 `(i-1, j)` 和正左方 `(i, j-1)` 发起路径数查询，并将两者返回的结果相加，作为当前坐标的总路径数返回。
* `return dfs(m - 1, n - 1)`
    * 在主函数中触发整个递归过程。由于网格大小为 $m \times n$，且坐标索引从 0 开始计数，因此右下角终点的坐标是 $(m-1, n-1)$。将终点坐标传入 `dfs` 函数开始倒推计算。

### （5）具体数值算例与执行表格

为便于展示递归全过程，设定一个极小的网格：**输入 $m = 2, n = 2$**。
目标是计算到达终点坐标 $(1, 1)$ 的路径数。由于启用了 `@cache`，相同的计算只发生一次。

| 执行顺序 | 状态与推导动作 | 数值与依赖关系 | 对应代码行 |
| :---: | :--- | :--- | :--- |
| 1 | **启动递归** ➔ 查询目标 $(1, 1)$ | `dfs(1, 1)` | `return dfs(m - 1, n - 1)` |
| 2 | 处理 $(1, 1)$ ➔ 需要上方和左方的数据 | `dfs(0, 1) + dfs(1, 0)` | `return dfs(i - 1, j) + dfs(i, j - 1)` |
| 3 | **展开** `dfs(0, 1)` ➔ 需要其上方和左方 | `dfs(-1, 1) + dfs(0, 0)` | `return dfs(i - 1, j) + dfs(i, j - 1)` |
| 4 | **执行** `dfs(-1, 1)` ➔ 行坐标 -1 越界 | 触碰边界，返回 `0` | `if i < 0 or j < 0: return 0` |
| 5 | **执行** `dfs(0, 0)` ➔ 到达起点 | 触碰起点，返回 `1` | `if i == 0 and j == 0: return 1` |
| 6 | **结算** `dfs(0, 1)` ➔ 汇总步骤 4 与 5 | $0 + 1 \Rightarrow$ **返回 `1`** (存入 cache) | `return dfs(i - 1, j) + dfs(i, j - 1)` |
| 7 | **展开** `dfs(1, 0)` ➔ 需要其上方和左方 | `dfs(0, 0) + dfs(1, -1)` | `return dfs(i - 1, j) + dfs(i, j - 1)` |
| 8 | **执行** `dfs(0, 0)` ➔ 到达起点 | 触碰起点，返回 `1` | `if i == 0 and j == 0: return 1` |
| 9 | **执行** `dfs(1, -1)` ➔ 列坐标 -1 越界 | 触碰边界，返回 `0` | `if i < 0 or j < 0: return 0` |
| 10 | **结算** `dfs(1, 0)` ➔ 汇总步骤 8 与 9 | $1 + 0 \Rightarrow$ **返回 `1`** (存入 cache) | `return dfs(i - 1, j) + dfs(i, j - 1)` |
| 11 | **最终结算** `dfs(1, 1)` ➔ 汇总步骤 6 与 10| $1 + 1 \Rightarrow$ **最终答案 `2`** | `return dfs(i - 1, j) + dfs(i, j - 1)` |
